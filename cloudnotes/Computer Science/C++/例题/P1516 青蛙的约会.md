
## 题目描述

两只青蛙在网上相识了，它们聊得很开心，于是觉得很有必要见一面。它们很高兴地发现它们住在同一条纬度线上，于是它们约定各自朝西跳，直到碰面为止。可是它们出发之前忘记了一件很重要的事情，既没有问清楚对方的特征，也没有约定见面的具体位置。不过青蛙们都是很乐观的，它们觉得只要一直朝着某个方向跳下去，总能碰到对方的。但是除非这两只青蛙在同一时间跳到同一点上，不然是永远都不可能碰面的。为了帮助这两只乐观的青蛙，你被要求写一个程序来判断这两只青蛙是否能够碰面，会在什么时候碰面。

我们把这两只青蛙分别叫做青蛙 A 和青蛙 B，并且规定纬度线上东经 $0$ 度处为原点，由东往西为正方向，单位长度 $1$ 米，这样我们就得到了一条首尾相接的数轴。设青蛙 A 的出发点坐标是 $x$，青蛙 B 的出发点坐标是 $y$。青蛙 A 一次能跳 $m$ 米，青蛙 B 一次能跳 $n$ 米，两只青蛙跳一次所花费的时间相同。纬度线总长 $L$ 米。现在要你求出它们跳了几次以后才会碰面。

## 输入格式

输入只包括一行五个整数 $x,y,m,n,L$。

## 输出格式

输出碰面所需要的次数，如果永远不可能碰面则输出一行一个字符串 `Impossible`。

## 输入输出样例 #1

### 输入 #1

```
1 2 3 4 5

```

### 输出 #1

```
4

```

## 说明/提示

对于 $100\%$ 的数据，$1 \le x, y, m, n \le 2 \times 10^9$，$x \ne y$，$1 \le L \le 2.1 \times 10^9$。


# 枚举 TLE

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    long long int x,y,m,n,L;
    long long int dist;
    long long int step;
    long long int count=0;
    cin >> x >>y >>m >>n >>L;

    if ( x<y && m<n){
        step= n-m;
        dist= L-y+x;
    } else if ( x>=y && m<n){
        step= n-m;
        dist= x-y;
    } else if ( x<y && m>=n){
        step = m-n;
        dist = y-x;
    } else{
        step = m-n;
        dist = L-x+y;
    }
    
     if (dist == 0) {
        cout << 0 << endl;
        return 0;
    }
    
    if (step == 0) {
        cout << "Impossible" << endl;
        return 0;
    }

    long long k = 0;
    const long long MAX_K = 700000000;
    while (k <= MAX_K) {
        if ((dist + k * L) % step == 0) {
            count = (dist + k * L) / step;
            cout << count << endl;
            return 0;
        }
        k++;
        if (k > L) { 
            long long g = __gcd(L, step);
            if (dist % g != 0) {
                cout << "Impossible" << endl;
                return 0;
            }
        }
    }
    cout << "Impossible" << endl;
    return 0;
}
 
```

# 最后的正解

```cpp
#include <iostream>
using namespace std;

long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) {
        x = 1; y = 0;
        return a;
    }
    long long g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}
int main() {
    long long x, y, m, n, L;
    cin >> x >> y >> m >> n >> L;

    long long a = m - n;
    long long b = y - x;
    
    if (a < 0) { a = -a; b = -b; }
    
    long long u, v;
    long long g = exgcd(a, L, u, v);
    
    if (b % g != 0) {
        cout << "Impossible" << endl;
        return 0;
    }
    
    long long mod = L / g;
    long long t = (u * (b / g)) % mod;
    if (t < 0) t += mod;
    
    cout << t << endl;
    
    return 0;
}

```

形式化题面：
判断是否存在一个最小的 k，使 x+km≡y+kn(modL)，并求这个最小的 k。

### 如何用 Exgcd 求解此题？

不妨设 m>n。

根据同余的性质，有 k(m−n)≡y−x(modL)，并且可以得出一个方程：k(m−n)+aL=y−x。使用 Exgcd 可以求解该方程的最小正整数解或判断其无解。

当 m<n 时，交换两只青蛙的信息即可。


（**≡** 是同余符号，读作"恒等于"或"同余于"。在数论中，它表示两个数除以同一个数有相同的余数。）
