
### 图像数字化的主要过程
涉及采样和量化两个步骤：
- 采样：把在空间上连续的图像转换成离散的采样点（像素）集的操作。
- 量化：把采样后得到的各像素点的颜色值从模拟量转换到离散量的过程。
**在量化过程中，图像的连续颜色值（即色彩的亮度和色度）被映射到有限的离散值，题目中特别强调是对颜色取值进行的操作，颜色取值是已经经过采样过程而得到的，因此是量化而非采样**。

### 像素坐标
　　在图像处理领域，通常以图像左上角为原点建立直角坐标系。一个像素的坐标为 (x,y)，表示它是从左往右数第 x 号、从上往下数第 y 号像素，x 和 y 都从 0 开始。例如，如果一幅图像的分辨率是 30×40，则左上角像素的坐标是 (0,0)，右下角像素的坐标是（29,39）。

### 混色模型

|模型类型|混色原理|适用场景|三原色|
|---|---|---|---|
|**相加混色模型 (RGB)**|**光的三原色**：不同颜色的光叠加，混合后亮度增加，最终可得到白色光。|显示屏、电视、投影仪等**自发光的设备**。|**R(红)、G(绿)、B(蓝)**|
|**相减混色模型 (CMY/K)**|**颜料的三原色**：颜料吸收（减去）特定波长的光，反射其余光。混合越多，吸收的光越多，反射越少，最终趋于黑色。|**印刷、打印、绘画**等依靠反射光显色的场景。|**C(青)、M(品红)、Y(黄)**|

### 颜色
(0,156,68)，表示这个颜色是由等级为 0 的红色、156 的绿色、68 的蓝色组成
RGB 颜色每个分量被分成 0~255 共 256 个等级，每个分量占一个字节，如等级 0 在计算机中的表示是 00000000，255 是 11111111。一个 RGB 颜色包含三个分量，占三个字节，如 (0,156,68) 在计算机中的表示是 00000000 10011100 1000100。

如下程序的功能是将 boy.jpg 图像所有像素的绿色和蓝色分量设置为 0，红色分量保持不变。
```python
from PIL import Image
boy = Image.open('src/step3/source/boy.jpg')
w, h = boy.size
for x in range(w):
    for y in range(h):
        r,g,b = boy.getpixel((x,y))
        boy.putpixel( (x,y), (r, 0, 0) )
boy.save('src/step3/student/boy2.jpg')
```
第 3 行：获取图像分辨率，w为宽度、h为高度；
第 4~5 行：利用嵌套循环形成所有可能的像素坐标(x, y)（访问图像所有像素与访问矩阵所有元素的方法是一致的，因为图像可以看作是一个矩阵，矩阵元素即像素中的颜色）；
第 6 行：获取(x, y)处像素的 RGB 颜色；
第 7 行：putpixel函数的功能是修改某个像素的颜色，此处是将(x, y)处像素的颜色修改为(r, 0, 0)，即只保留红色分量；
第 8 行：将修改结果保存为图像文件。

### RGBA颜色
　　在 RGB 颜色的基础上，再加入一个 A 分量，就构成了 RGBA 颜色。A 分量表示的是像素透明度，取值也是 0~255，0 表示完全透明，255 表示完全不透明。
下面程序的功能是将 boy.jpg 图像所有像素的透明度均改为 100，从而使整幅图像呈现半透明效果。
```python
from PIL import Image
boy = Image.open('src/step4/source/boy.jpg')
boy = boy.convert('RGBA')
w, h = boy.size
for x in range(0, w):
    for y in range(0, h):
        r, g, b, a = boy.getpixel((x, y))
        a = 100
        boy.putpixel((x, y), (r, g, b, a))
boy.save('src/step4/student/boy2.png')
```
　关于此程序，有几个地方需要说明：
　　1）jpg 图片采用的是 RGB 颜色模式，不包含表示透明度的 A 分量，而程序第 3 行的convert('RGBA')函数是将该图像转换成 RGBA 模式，转换之后，每个像素会增加一个 A 分量，变成 RGBA 颜色，而新增的 A 分量取值是 255，完全不透明。
　　2）程序第 7~9 行的功能是：将某个像素的 RGBA 颜色提取出来（第 7 行）；将 A 分量改为 100，也就是将这个像素设置为半透明（第 8 行）；最后再将修改后的 RGBA 颜色重新放回原来的像素（第 9 行）；
　　3）程序最后一行是将修改后的图像保存为文件，需注意的是，jpg 格式一般不支持透明度，如果保存为 jpg 格式，则之前设置的透明度信息就会丢失，所以这里保存为 png 格式，这种格式支持透明度；

### 抠图原理
初始代码给出了绿幕抠像的程序框架，与上关代码基本一样，区别在于：上关是将所有像素都改为半透明，而本关只对偏绿色像素进行修改（改为完全透明）。所以，只要在第 9 行问号处给出偏绿色像素的判断条件，即可过关。

　　绿色分量占的比例可以用绿色分量除以三个分量之和来表示，即 (G+1)/(R+G+B+3)，这里给每个分量均增加 1，是为了避免 R、G、B 分量均为 0 时发生除 0 错误。根据视觉效果可以看出，当绿色分量的比例超过 40% 时，颜色看起来就是偏绿色了。当绿色分量取值较小的时候，颜色看起来是黑色，当超过一定数值时看起来才是绿色。　　所以，在“比例超过 40%”这个条件的基础上，还要再加一个条件，绿色分量的取值要大于某一个值，根据视觉效果，这个值可以设置为 60，即在程序第 9 行中增加条件 G>60。
```python
from PIL import Image
boy = Image.open('src/step5/source/boy.jpg')
boy = boy.convert('RGBA')
w, h = boy.size
for x in range(0, w):
    for y in range(0, h):
        r, g, b, a = boy.getpixel((x, y))
        if (g+1)/(r+g+b+3) > 0.4 and g>60:  #如果是偏绿色
            a = 0
            boy.putpixel((x, y), (r, g, b, a))
boy.save('src/step5/student/boy2.png')
```

### Image 模块函数介绍
- img.rotate(a)：
将img图像顺时针旋转 a 度；
- img.resize((w, h))：
缩小或放大img图像，缩放后的分辨率为 w×h；
- img.crop((x1,y1,x2,y2))：
从img图像中的截取一个矩形区域，矩形左上角像素坐标为 (x1,y1)，右下角坐标为 (x2,y2)；
- img.paste(img2, (x,y))：
将img2图像粘贴到img中，粘贴后，img2左上角像素在img中的坐标是 (x,y)。

```python
from PIL import Image #导入PIL库中的Image模块  
boy = Image.open('src/step6/source/boy2.png')  #之前抠取的小男孩图像  
playground = Image.open('src/step6/source/playground.jpg')#背景图像  
w, h = boy.size  #获取小男孩图像的宽和高  
boy = boy.resize((w//2, h//2)) #将小男孩图像的宽和高缩放到原来的1/2  
playground.paste(boy, (620,140), mask=boy.split()[3]) #将小男孩粘贴到背景中  
playground.save('src/step6/student/final.jpg')  #将最终图像保存为文件  
```

程序第 7 行的mask=boy.split()[ 3 ]是在设置粘贴时使用的透明度分量，若删去该部分，则粘贴时不使用透明度。
