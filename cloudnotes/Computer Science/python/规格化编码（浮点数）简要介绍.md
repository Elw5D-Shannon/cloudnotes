

## 一、基本概念
规格化编码是浮点数表示中的标准化方法，目的是用固定长度的二进制位最大限度地提高数值的精度和表示范围。

## 二、核心原理

### 2.1 二进制科学计数法
一个二进制数表示为：
$$
\pm M \times 2^{E}
$$
其中：
- **M**：尾数（mantissa），通常是绝对值在 $[1, 2)$ 或 $[0.5, 1)$ 范围内的小数
- **E**：阶码（exponent），整数

### 2.2 隐藏位技术
对于二进制规格化数，尾数的最高位总是 1（如 `1.xxxxx`），因此可以不存储这个 1，从而多一位精度（IEEE 754 标准做法）。

## 三、IEEE 754 单精度格式

### 3.1 存储结构

1位符号S-8位阶码E-23位尾数域F

### 3.2 规格化条件
- 阶码 $E \neq 0$ 且 $E \neq 255$（全1）
- 实际尾数值 $M = 1.F$（隐含1）
- 阶码使用移码表示（偏移量 = 127）

## 四、十进制转规格化浮点数示例

### 示例：将 $-13.625$ 转换为 IEEE 754 单精度格式

#### 步骤 1：转换为二进制
- 整数部分：$13_{10} = 1101_2$
- 小数部分：
  ```
  0.625 × 2 = 1.25 → 1
  0.25  × 2 = 0.5  → 0
  0.5   × 2 = 1.0  → 1
  ```
  结果：$0.625_{10} = 0.101_2$
- 合并：$13.625_{10} = 1101.101_2$

#### 步骤 2：二进制规格化
移动小数点：
$1101.101_2 = 1.101101_2 \times 2^{3}$
- 尾数 $M = 1.101101$
- 阶码 $E = 3$

#### 步骤 3：计算阶码（加偏移）
偏移量 = 127
存储阶码 = $3 + 127 = 130_{10} = 10000010_2$

#### 步骤 4：提取各部分
- 符号位 $S = 1$（负数）
- 尾数域 $F = 101101$（后补零至23位）：
  `10110100000000000000000`

#### 步骤 5：组合结果
```
S    E              F
1    10000010    10110100000000000000000
```
十六进制：`C15A0000`

## 五、不同编码方式的规格化

| 编码方式 | 正数规格化形式 | 负数规格化形式 | 说明 |
|---------|--------------|--------------|------|
| **原码尾数** | `0.1xxx…` | `1.1xxx…` | 最高数值位总是1 |
| **补码尾数** | `0.1xxx…` | `1.0xxx…` | 负数最高数值位为0 |
| **IEEE 754** | 隐含`1.xxx…` | 隐含`1.xxx…` | 尾数为原码绝对值 |

## 六、关键特性总结

1. **精度优化**：隐藏的最高位1使23位尾数域实际表示24位精度
2. **范围扩展**：阶码使用移码便于比较和运算
3. **特殊值**：
   - 阶码全0：非规格化数（包括0）
   - 阶码全1：无穷大或NaN
4. **标准化**：统一格式便于不同系统间数据交换
