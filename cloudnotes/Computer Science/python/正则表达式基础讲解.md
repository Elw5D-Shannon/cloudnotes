## 一、核心基础语法

正则的核心是元字符（有特殊含义的符号）+ 普通字符（字母、数字等无特殊含义的字符），先记最常用的核心元字符，按功能分类更易记。

### 1. 匹配单个字符

| 语法 | 含义 | 示例 |
| :--- | :--- | :--- |
| `[0-9]` | 匹配单个数字（等价 `\d`） | `\d` 匹配 `5`，不匹配 `a` |
| `[a-z]` | 匹配单个小写字母 | `[a-z]` 匹配 `s`，不匹配 `S` |
| `[A-Z]` | 匹配单个大写字母 | `[A-Z]` 匹配 `P`，不匹配 `p` |
| `[a-zA-Z0-9]` | 匹配字母 + 数字（等价 `\w`） | `\w` 匹配 `9`/`a`/`Z`，不匹配 `@` |
| `.` | 匹配任意单个字符（除了换行 `\n`） | `.` 匹配 `a`/`5`/`@`，不匹配 `\n` |
| `[^0-9]` | 匹配非数字（`^` 在 `[]` 内表示 “取反”） | `[^0-9]` 匹配 `a`/`@`，不匹配 `8` |
| `[]` | 匹配 `[]` 内任意一个字符 | `[abc]` 匹配 `a`/`b`/`c`，不匹配 `d` |
### 2. 匹配数量（限定前面字符的出现次数）

| 语法 | 含义 | 示例 |
| :--- | :--- | :--- |
| `*` | 0 次或多次（尽可能多匹配） | `a*` 匹配 `""`/`a`/`aaa` |
| `+` | 1 次或多次 | `a+` 匹配 `a`/`aaa`，不匹配 `""` |
| `?` | 0 次或 1 次（可选） | `a?` 匹配 `""`/`a`，不匹配 `aa` |
| `{n}` | 恰好 n 次 | `a{3}` 匹配 `aaa`，不匹配 `aa`/`aaaa` |
| `{n,}` | 至少 n 次 | `a{2,}` 匹配 `aa`/`aaa`，不匹配 `a` |
| `{n,m}` | n 到 m 次（包含 n 和 m） | `a{2,4}` 匹配 `aa`/`aaa`/`aaaa` |
### 3. 匹配位置（锚点，不匹配具体字符，只匹配位置）

| 语法 | 含义 | 示例 |
| :--- | :--- | :--- |
| `^` | 匹配字符串开头（`[]` 外） | `^a` 匹配 `abc` 的 `a`，不匹配 `bac` 的 `a` |
| `$` | 匹配字符串结尾 | `c$` 匹配 `abc` 的 `c`，不匹配 `acb` 的 `c` |
| `\b` | 匹配单词边界（字母/数字与非字母/数字的分界） | `\bis\b` 匹配 `this is` 的 `is`，不匹配 `this` 的 `is` |
### 4. 转义字符（`\`）

如果要匹配元字符本身（比如 `./*/(`），需要在前面加 `\` 转义，因为这些字符在正则中有特殊含义。

**示例**：
- 匹配小数点 `.`：需要写 `\.`（直接写 `.` 会匹配任意字符）
- 匹配星号 `*`：需要写 `\*`
- 匹配括号 `(`：需要写 `\(`

### 5. 逻辑匹配

| 语法 | 含义 | 示例 |
| :--- | :--- | :--- |
| <code>\|</code> | 或（匹配左边或右边） | <code>a\|b</code> 匹配 `a` 或 `b`；<code>\d\|a</code> 匹配数字或小写 `a` |
| `()` | 分组（将多个字符视为一个整体，也可用于提取） | `(ab)+` 匹配 `ab`/`abab`（`ab` 整体出现 1 次 +） |
## 二、常用修饰符（模式修正符）

修饰符用于调整正则的匹配规则，写在正则表达式的末尾，不同语言中写法略有差异
（比如 Python 是 `re.compile(r"xxx", re.I)`，JS 是 `/xxx/i`），核心常用修饰符：

| 修饰符 | 含义 | 简称（Python/JS） |
| :--- | :--- | :--- |
| 忽略大小写 | 匹配时不区分字母大小写 | `i`（`re.I` / `/i`） |
| 多行模式 | `^` 匹配每行开头，`$` 匹配每行结尾 | `m`（`re.M` / `/m`） |
| 单行模式 | 让 `.` 匹配所有字符（包括换行 `\n`） | `s`（`re.S` / `/s`） |
| 全局匹配 | 匹配所有符合规则的结果，而非只匹配第一个 | `g`（JS 专属，Python 用 `re.findall` 实现） |
## 三、实战经典示例

### 示例 1：验证手机号（11 位，以 13/14/15/17/18/19 开头）

**规则分析**：
- 开头固定：`1`，后跟 `3/4/5/7/8/19` → `1[345789]`
- 后面 9 位都是数字 → `\d{9}`
- 整串必须是 11 位（开头到结尾无多余字符）→ 加 `^` 和 `$`

**正则表达式**：`^1[345789]\d{9}$`

### 示例 2：验证邮箱（基础版，适配 99% 常见邮箱）

**规则分析**：
- 前缀：字母 / 数字 / 下划线 / 点，1 次 + → `\w+(\.\w+)*`
- `@` 符号：固定 `@`
- 域名：字母 / 数字 / 点 / 横线，1 次 + → `[a-zA-Z0-9-]+(\.[a-zA-Z0-9-]+)*`
- 后缀：`.com`/`.cn`/`.net` 等，2-4 位字母 → `\.[a-zA-Z]{2,4}`

**正则表达式**：`^\w+(\.\w+)*@[a-zA-Z0-9-]+(\.[a-zA-Z0-9-]+)*\.[a-zA-Z]{2,4}$`

### 示例 3：提取字符串中的所有数字

**正则**：`\d+`（`+` 表示 1 次 +，避免提取单个 0/1，比如提取 `123` 而非 `1`/`2`/`3`）

**示例文本**：`张三今年28岁，月薪15000元，身高180cm`

**提取结果**：`28`、`15000`、`180`

### 示例 4：验证身份证号（18 位，最后一位可能是 X/x）

**规则分析**：
- 前 17 位：纯数字 → `\d{17}`
- 最后 1 位：数字或 `X`/`x` → `[\dXx]`
- 整串 18 位 → 加 `^` 和 `$`

**正则表达式**：`^\d{17}[\dXx]$`

### 示例 5：验证密码强度（8-16 位，包含字母 + 数字）

**规则分析**：
- 长度 8-16 位 → 整体 `^.{8,16}$`
- 包含至少 1 个字母 → `(?=.*[a-zA-Z])`（正向预查）
- 包含至少 1 个数字 → `(?=.*\d)`

**正则表达式**：`^(?=.*[a-zA-Z])(?=.*\d).{8,16}$`
> **注**：正向预查 `(?=...)` 只做判断，不占匹配字符，用于 “必须包含某类字符” 的校验。
