**DMA（直接存储器访问，Direct Memory Access）** 是一种计算机技术，允许**外部设备或子系统**在不通过中央处理器（CPU）的情况下，**直接与系统内存进行数据交换**。这能**大幅减轻CPU的负担**，提升系统整体性能。

### 操作系统简述
操作系统（OS）是管理计算机软硬件资源并向用户提供公共服务的软件。
- 进程管理
- 存储管理 
- 文件管理 
- 设备管理
操作系统是计算机系统中最基础的系统软件，而不是应用软件。系统软件包括操作系统、驱动程序、工具软件等，负责管理计算机硬件资源和提供基本服务。应用软件是用户为实现特定任务而使用的程序，如文字处理软件、浏览器、游戏等。因此，操作系统是系统软件，不是应用软件。

CPU能够直接执行的是机器指令。机器指令是计算机硬件能够直接理解和执行的指令，它们通常以二进制形式存在。汇编指令需要通过汇编程序转换成机器指令后，CPU才能执行。而Python语句是高级编程语言的代码，需要通过Python解释器转换成机器指令后，CPU才能执行。

## 分页式存储管理介绍
- 将**主存（物理内存）**划分为固定大小的块，称为**页框（Frame）**。
- 将**进程的逻辑地址空间**也划分为相同大小的块，称为**页（Page）**。
- 操作系统维护一张**页表（Page Table）**，记录每个逻辑页号到物理页框号的映射。
### 地址转换
计算机内存中每个字节，都被赋予一个唯一的序号，称为地址
逻辑地址 = 页号 + 页内偏移量  
页内偏移量位数由页大小决定。
例：页大小 = 32 MB = 225225 字节 → 页内偏移占 25 位。  
若逻辑地址是 32 位，则页号占 32−25=732−25=7 位 → 最多 27=12827=128 页。
CPU 访问逻辑地址时：
1. 用页号查页表得到物理页框号。
2. 物理地址 = 物理页框号 × 页大小 + 页内偏移。

一个存储容量为16KB的存储器，按字节编址，其地址总线的宽度最少为（  ）。
$16KB=16*2^{10}B=2^{14}B$，最少14根地址总线

1. **TCP/IP 模型** 分为四层（或五层，若细分数据链路层与物理层）
    - 应用层（Application Layer）
    - 传输层（Transport Layer）
    - 网络层（Internet Layer）
    - 网络接口层（Network Interface Layer）
2. **ISO/OSI 模型** 分为七层
    - 应用层（Application）
    - 表示层（Presentation）
    - 会话层（Session）
    - 传输层（Transport）
    - 网络层（Network）
    - 数据链路层（Data Link）
    - 物理层（Physical）
3. **对应关系**：
    - TCP/IP 的**应用层** 整合了 OSI 的 **应用层、表示层、会话层** 的功能。
    - TCP/IP 的**传输层** 对应 OSI 的**传输层**。
    - TCP/IP 的**网络层** 对应 OSI 的**网络层**。
    - TCP/IP 的**网络接口层** 对应 OSI 的**数据链路层 + 物理层**。
