### 进程和程序的关系
- 进程是动态的，程序是静态的 
- 进程有生命周期，会消亡，关机后不保存，而程序可长期保存在外存上 
- 一个程序的多次运行对应到多个进程 
  一个进程可以通过调用运行多个程序
### 进程的组成
- 程序
- 数据
- 进程控制块（PCB，process control block）
```python
import psutil 
for p in psutil.process_iter(): 
    print(p.pid)         #进程号
    print(p.name())      #进程名
    print(p.status())     #状态
    print(p.username())    #用户名
    print(p.cpu_percent()) #cpu占用率
    print('-'*10)
# 返回系统中所有正在运行的进程
```

关于进程控制块（PCB）：
进程控制块是操作系统（OS）为了管理和控制进程的运行，创建的记录进程信息的表 
PCB是进程存在的唯一标志，随着进程的创建而建立，随着进程的撤消而消失 
在进程生命周期中，OS利用PCB对进程进行调度和资源分配
### 进程的状态：
- 运行：进程在CPU上执行
- 堵塞：进程获得了除CPU之外 的一切所需资源
- 就绪：进程等待某一事件完成 而暂停运行
**某一时刻，一个进程有且仅有一种状态**

<u>注意：堵塞不能变为运行，就绪不能变为堵塞</u>

进程状态转换并不需要一一对应。 阻塞转变为就绪，可能只是因为等待的 I/O 完成或资源可用，此时就绪队列增加一个进程，但 CPU 可能继续运行当前进程，不一定立即发生就绪 → 运行的切换。
<u>所以“当一个进程从阻塞状态变成就绪状态, 则一定有一个进程从就绪状态变成运行状态。”的说法是错误的</u>

**单核 CPU 计算机，最多有一个进程处于运行状态，但可有多个进程处于就绪状态**

### 进程调度：

#### **多级反馈队列
是一种动态优先级调度算法，结合了多种调度策略（如优先级调度、时间片轮转）的特点，能根据进程的历史行为动态调整其优先级。
1. **设立多个就绪队列**，每个队列有不同的优先级。
2. **高优先级队列**的时间片较短（适合交互式短进程，响应快）。 
3. **低优先级队列**的时间片较长（适合 CPU 密集型长进程，减少切换开销）。
4. **进程可在队列间移动**：
    - 如果一个进程用完时间片还没结束，则降低优先级（移到下一级队列）。
    - 如果一个进程在时间片用完前主动放弃 CPU（如等待 I/O），则保持或提高优先级（防止长作业饥饿交互式作业）。
