
## 字符变数字！
```cpp
字符'0'的ASCII码是48
字符'1'的ASCII码是49
...
字符'9'的ASCII码是57

所以：
'5' - '0' = 53 - 48 = 5
'9' - '0' = 57 - 48 = 9
'0' - '0' = 48 - 48 = 0
```


## 如果C语言中字符整型数据小于0或超过127，会打印出什么字符？

我们知道，ascii码表（部分）如下：

|                  |                  |                  |                   |                        |      |
| ---------------- | ---------------- | ---------------- | ----------------- | ---------------------- | ---- |
| Bin<br><br>(二进制) | Oct<br><br>(八进制) | Dec<br><br>(十进制) | Hex<br><br>(十六进制) | 缩写/字符                  | 解释   |
| 0000 0000        | 00               | 0                | 0x00              | NUL(null)              | 空字符  |
| 0000 0001        | 01               | 1                | 0x01              | SOH(start of headline) | 标题开始 |
| 0000 0010        | 02               | 2                | 0x02              | STX (start of text)    | 正文开始 |
| **...**          |                  |                  |                   |                        |      |
| 0111 1110        | 0176             | 126              | 0x7E              | ~                      | 波浪号  |
| 0111 1111        | 0177             | 127              | 0x7F              | DEL (delete)           | 删除   |

字符整型数据取值为[0,127]，那么**如果字符编码小于0或者大于127的话，会打印什么字符？又是如何处理的？**


当字符编码属于257~288范围时，开始出现­，因为ASCII码前32位是控制字符或通信专用字符，无特定图形显示，或许根据使用编译器不同而有不同格式显示，在Dev-C++ 5.11版本中部分显示为­，部分无法打印出结果。

当字符编码属于289~384范围时，从感叹号开始顺序排列。即char(289)对应 ！ ，char(384)对应 € 。但因为char(382)对应 ~ ，所以char(383)应对应删除控制符，在Dev-C++ 5.11版本中无法打印结果，至于为什么char(385)打印结果为 €，尚不清楚。

综上，当字符编码n大于等于0时，若（n%256） <127 ，则正常输出char(n%256)对应字符 ，若n%256>=127,则不能被正常打印。

当字符编码n小于0时，若（256+n%256 ）<127.   则正常输出char(256+n%256)对应字符 ，若（256+n%256）>=127,则不能被正常打印。


## 字符转二进制

```cpp
#include <iostream>
using namespace std;
int main() {
    char ch;
    
    // 输入一个字符
    cin >> ch;
    
    // 获取字符的ASCII码
    int ascii = (int)ch;
    
    // 转换为8位二进制并输出
    for (int i = 7; i >= 0; i--) {
        // 通过位运算获取每一位
        int bit = (ascii >> i) & 1;
        cout << bit;
    }
    
    cout << endl;
    
    return 0;
}
```
